# Security Policy

## Supported versions

Security fixes are provided for the latest version on the default branch.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Report it
privately through the repository's GitHub Security Advisories page, including:

- A concise description of the issue and affected area.
- Steps to reproduce or a proof of concept.
- The potential impact and any suggested mitigation.

Reports are acknowledged within five business days. A fix, mitigation, or
status update will follow after triage.
