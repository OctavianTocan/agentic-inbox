# skill-gen (agentic-inbox)

Tool: `tools/skill-gen/`. Commands: `bun run skills:generate` / `skills:check`.

## Markers

Embed fragments next to the code they describe:

```ts
//<skill-gen>
// ---
// name: domain-backend
// description: "…"
// ---
//
// ## Section
// …
//</skill-gen>
```

Same `name` merges across files into `.agent/skills/<name>/SKILL.md`
(`<!-- AUTO-GENERATED by skill-gen -->`). Hand-written skills are not overwritten.
`references/` stays hand-maintained.

## Conventions here

- Primary skill: `domain-backend` (fragments in Layers, Repos, DecodeSqlRow, Api, inbox client, …).
- TSDoc on the export may sit **above** the skill-gen block — both are fine.
- Do not hand-edit generated `SKILL.md`; edit source fragments + regenerate.
- Registry `_index.md` / `_manifest.jsonl` are **not** owned by skill-gen.

See `tools/skill-gen/README.md`, `tools/AGENTS.md`.
